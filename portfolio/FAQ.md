## 🎨 أسئلة التصميم

### س: كيف أغير الألوان؟
**ج:** افتح `styles.css` وابحث عن `:root`، ثم عدّل:
```css
--primary-color: #لونك-الجديد;
--secondary-color: #لونك-الثانوي;
```

### س: كيف أغير الخطوط؟
**ج:** 
1. اختر خط من [Google Fonts](https://fonts.google.com/)
2. أضف رابط الخط في `<head>`
3. عدّل في `styles.css`:
```css
--font-arabic: 'اسم-الخط-الجديد', sans-serif;
```

### س: كيف أضيف صورتي الشخصية؟
**ج:**
1. ضع صورتك في مجلد `images/`
2. سمّها `profile.jpg`
3. أو عدّل المسار في HTML:
```html
<img src="images/your-photo.jpg" alt="اسمك">
```

### س: الصورة لا تظهر، ما الحل؟
**ج:** تأكد من:
- المسار صحيح: `images/profile.jpg`
- الصورة موجودة في المجلد
- اسم الملف مطابق تماماً (حساس لحالة الأحرف)
- صيغة الصورة مدعومة (jpg, png, webp)

### س: كيف أضيف/أحذف قسم؟
**ج:**
- **لإخفاء قسم:** أضف في CSS:
```css
#section-id { display: none; }
```
- **لإضافة قسم:** انسخ هيكل قسم موجود وعدّل المحتوى

## 💻 أسئلة تقنية

### س: كيف أختبر البورتفوليو قبل النشر؟
**ج:** 
1. افتح `index.html` في المتصفح مباشرة
2. أو استخدم Live Server في VS Code
3. اختبر على أجهزة مختلفة

### س: كيف أنشر البورتفوليو؟
**ج:** أسهل طريقة:
1. أنشئ حساب على [GitHub](https://github.com/)
2. أنشئ repository باسم `username.github.io`
3. ارفع الملفات
4. موقعك سيكون: `https://username.github.io`

راجع قسم "نشر البورتفوليو" في `README.md` للتفاصيل.

### س: هل أحتاج استضافة مدفوعة؟
**ج:** لا، يمكنك استخدام:
- **GitHub Pages** - مجاني
- **Netlify** - مجاني
- **Vercel** - مجاني

كلها توفر HTTPS مجاناً.

### س: كيف أربط Domain خاص؟
**ج:**
1. اشترِ Domain من (Namecheap, GoDaddy, etc.)
2. في إعدادات GitHub Pages/Netlify، أضف Domain
3. عدّل DNS settings في موقع Domain
4. انتظر 24-48 ساعة للتفعيل

### س: الموقع بطيء، كيف أحسّن السرعة؟
**ج:**
1. ضغط الصور باستخدام [TinyPNG](https://tinypng.com/)
2. استخدم WebP format
3. فعّل Lazy Loading
4. صغّر CSS و JS (minify)
5. استخدم CDN

راجع قسم "تحسين الأداء" في `ADVANCED_TIPS.md`.

## 📝 أسئلة المحتوى

### س: كم مشروع يجب أن أضيف؟
**ج:** 4-8 مشاريع قوية أفضل من 20 مشروع متوسط. اختر:
- مشاريع حقيقية لعملاء
- مشاريع شخصية مبتكرة
- مشاريع توضح مهارات متنوعة

### س: ليس لدي خبرة عملية، ماذا أفعل؟
**ج:**
- أضف مشاريع شخصية
- مشاريع تطوعية
- مشاريع تدريبية (إن كانت قوية)
- مشاريع Freelance
- مشاريع أكاديمية مميزة

### س: كيف أكتب إنجازاتي بشكل مؤثر؟
**ج:** استخدم صيغة STAR:
- **S**ituation: الموقف
- **T**ask: المهمة
- **A**ction: الإجراء
- **R**esult: النتيجة (بأرقام!)

**مثال:**
❌ "عملت على تحسين الموقع"
✅ "قادت إعادة تصميم الموقع مما زاد التحويل بنسبة 45%"

راجع `EXAMPLES.md` للمزيد من الأمثلة.

### س: كيف أحدد نسبة إتقان المهارات؟
**ج:**
- **90-100%:** خبير، تستخدمها يومياً
- **75-89%:** متقدم، مرتاح جداً
- **60-74%:** متوسط، تستخدمها بانتظام
- **أقل من 60%:** لا تضعها

كن صادقاً! المبالغة تضر أكثر مما تنفع.

### س: هل أضع السيرة الذاتية في البورتفوليو؟
**ج:** نعم، أضف زر "تحميل السيرة الذاتية" يربط بملف PDF محدث.

## 📧 أسئلة نموذج التواصل

### س: نموذج التواصل لا يعمل، ما الحل؟
**ج:** النموذج يحتاج إعداد خدمة إرسال. الخيارات:

**1. EmailJS (مجاني وسهل):**
- سجل في [EmailJS](https://www.emailjs.com/)
- اتبع التعليمات في `README.md`

**2. Formspree (أسهل):**
- سجل في [Formspree](https://formspree.io/)
- عدّل form tag:
```html
<form action="https://formspree.io/f/YOUR_ID" method="POST">
```

**3. Backend خاص:**
- إذا كان لديك سيرفر، أنشئ API endpoint

### س: كيف أحمي النموذج من Spam؟
**ج:**
- أضف Google reCAPTCHA
- استخدم Honeypot field
- فعّل Rate Limiting
- استخدم خدمة مثل Formspree (حماية مدمجة)

## 📱 أسئلة الموبايل

### س: التصميم مكسور على الموبايل، ما الحل؟
**ج:** تأكد من:
1. وجود viewport meta tag:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
2. عدم تعديل CSS بشكل خاطئ
3. اختبار على أجهزة حقيقية
4. فتح Console للتحقق من الأخطاء

### س: كيف أختبر على الموبايل بدون جهاز؟
**ج:**
1. **Chrome DevTools:** اضغط F12 > Toggle Device Toolbar
2. **Firefox Responsive Design Mode:** Ctrl+Shift+M
3. **أدوات أونلاين:** [BrowserStack](https://www.browserstack.com/)

### س: القائمة (Menu) لا تعمل على الموبايل؟
**ج:** تأكد من:
- JavaScript يعمل (افتح Console)
- لا توجد أخطاء في الكود
- الأزرار قابلة للنقر (حجم كافٍ)
